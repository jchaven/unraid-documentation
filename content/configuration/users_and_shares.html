Users and Shares

To access the data in the storage devices users and shares must be created. The creation of users and shares is pretty straightforward requiring minimal information and once created very little maintenance.

Users

Unraid has a built-in administrator account named "root" that must be used to manage the system. Currently, it is not possible to create additional administrator accounts or disable the root account. End-users are additional accounts to which permissions may be applied to shares and are created on the "Users" tab in the Unraid management interface. Shares

Unraid supports the creation of "user shares" and "disk shares". Disk shares are not used. User shares have been created to store files and the data exists on all disks. The following table lists the user shares that have been created.

Permissions

Currently, all users have full permissions to all SMB shares. The Kodi boxes have unchecked access to the NFS shares.

File Sharing Protocols

Unraid is based on Linux which uses NFS (Network File System) natively. However, support for the Windows sharing protocol SMB is supported with the built-in software SAMBA. This page will not go into detail on these protocols. Basically, NFS is used to share the content of Media shares with the Kodi boxes (also running Linux), while SMB is used to securely share the non-Media shares (Adam, Donna, Home, etc.) with Windows computers.
