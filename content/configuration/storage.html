Storage

The primary role of the server is storage. Creating a location central to all computers on the network (clients) to which those computers can save important files and documents. Files being centralized and off the client computer reduces the potential loss due to disk failure or mistake.

Array

The storage array is a collection of hard drives (disks) that when combined create a large single storage area for files. The data in array is protected by the use of at least one parity drive. If there are three hard drives in the array; one must be used for parity, the remaining drives can be used for data.

Should the parity drive fail the server will operate normally with data supplied to the clients coming from the data drives as usual. Once replaced, the parity drive will be rebuilt by Unraid automatically. Should a data drive fail then again nothing noticeable on the client will occur. The data from the failed data drive will be replicated and served from the parity drive. Once replaced, the data drive will be rebuilt automatically by Unraid using the information stored on the parity drive.

If a drive fails replace it as soon as possible. If the parity drive fails AND a data drive fails the information on the failed data drive will be lost.

Unlike typical RAID servers Unraid does not require that all drives in the array be the same size. This is the power of Unraid, disks can be different sizes allowing you to use older, smaller drives until they die. The only rule with Unraid is the storage size of the parity drive must be at least as large as the largest data drive. For example, if the server has three data drives consisting of a 2TB disk, a 3TB disk, and a 4TB disk; then the parity drive must be at least 4TB in size.

Cache Pools

Unraid supports the use of cache disks which can be collected into "pools" creating a single cache storage area. The purpose of caching is to make reading or writing data to the server faster by writing to storage outside of the array. Typically cache drives are SSDs being much quicker than the HDDs that comprise the array. At predetermined intervals Unraid will move the data to the array removing it from cache.

Cahce drives are not used on these file servers.

SATA Disks

The hard drives in the array are standard SATA drives that are 3.5 inch (typical desktop size). The drives are all designed for use in NAS servers that are expected to run for long durations and incur many reads and writes. Unraid does support "non-NAS" drives typically used in desktop computers. However, those drives may fail earlier than drives designed for use in servers.

Since the drives in the servers are all large (10TB or more) they have been selected to reduce noise and power consumption. When possible drives are Helium-filled which has been shown to produce less vibration and use less power.

SSD Disks

Currently, no SSD drives are used. When SSD drives are used investigate drives with SSD Power Loss Protection (PLP) built-in.

USB Boot Disk

The Unraid OS boots from a USB flash device located inside the server. The flash drive is a higher quality drive from a name-brand manufacturer (Sandisk, PNY). Experience has shown that using a lower-quality flash drive results in drive failure.

Unraid OS is licensed using the GUID of the flash drive. The license is transferrable to another flash drive but, is limited to a specific number of transfers and must be requested manually from Lime Technology.

Unassigned Devices

Through the use of the plugin "Unassigned Devices" Unraid can support additional drives that exist outside of the array or cache pools. This is handy for creating a "scratch" disk for temporary files without the overhead of parity and whose loss should the drive fail not have a great impact.

Another feature of Unassigned Devices is the ability to connect an external hard drive and share it on the network. This especially useful for maintaining backups.
