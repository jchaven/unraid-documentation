Maintaining File Server / NAS

The file server requires minimal maintenance and can run for months without intervention. The following image shows the key points on the Main tab on the Unraid management interface of which to monitor.

Unraid OS Update If there is an update for the Unraid operating system a badge will appear beside the version number at the top left.
Alerts indicator At the top right of the page is an icon that turns different colors based on the type of alerts have been generated.
Disk errors If there are any errors on drives in the array they will be displayed in the "Errors" column. Disks with errors should be replaced as-soon-as-possible..
Disk Usage The last two columns on the array listing show how much disk space has data on it and how much free space is left. As a rule of thumb I like to keep usage below 85%. This is done by replacing smaller disks with larger ones as needed (or when a disk fails).
Array Status At the lower left Unraid displays the current status of the array and any array operations (like parity check, disk rebuild, etc.) if one is running.
Temperatures The temperatures for the processor and the main board are displayed at the bottom in the center. Normal operating temperature is about 40c or below. Also on this page are the disk temperatures which should be around 40c depending on the disk and running disk operations.

Updates

Most maintenance comes from applying updates to plugins, containers, and the OS itself. Updating the Unraid operating system is done rarely - about once per year - and only after a release has been available for several weeks.

Before applying updates to Tailscale or Unraid OS schedule a reboot ~30 minutes ahead using the at utility. This should allow you to reconnect to the server if something goes wrong.

Updating Unraid OS

When a new version of Unraid is available an orange colored badge will appear at the top-left corner of the management interface. Clicking on this badge displays the changes in the new version and a link to start the upgrade.

If the badge is no-longer displayed (by dismissing or inadvertently choosing to ignore) the upgrade can be started by clicking on the Tools tab and then clicking on Upgrade OS at the bottom.

Virtual Machines

There is nothing to do for Virtual machines. Any updates to the VM system are part of the OS. Updates to the guest machine's operating system are not handled by Unraid.

Parity Checks

Parity checks make sure the information on the parity drive is accurate and can emulate the data on a drive should one fail. Unraid is configured to perform a parity check once per month. If the automated parity check detects issues it will not write corrections to parity. At this point a manual parity check should be run.

Replacing Disks

Should a drive fail or show signs of impending failure (increasing error counts) it should be replaced ASAP.
